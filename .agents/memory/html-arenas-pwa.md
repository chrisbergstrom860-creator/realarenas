# PWA layer (shipped)

Pieces: dynamic BASE-aware `manifest.webmanifest` route + `/icons/:file` (whitelist) + `sw.js` route (no-cache, `__SUPABASE_HOST__` substituted at serve time) in server.js ("── PWA:" block); `html/sw.js`; `html/arenas-pwa.js` (SW registration + install card, self-computes BASE); `html/arenas-offline.html`; icons generated by `scripts/generate-icons.js` from `html/arenas-icon.svg` (drawn track glyph, NOT the emoji). Brand: theme `#FFD21E`, background `#F9FAFB`.

## Rules

**Never cache authenticated navigations in the SW.** Only `PUBLIC_PAGES` (landing/about/terms/privacy/for-clubs/how-points-work/offline) may be cache-fallback targets for navigations; app pages fall back to the offline page.
**Why:** server-rendered pages bake the signed-in user's data into HTML — a cached `/feed` can replay user A's page to user B on a shared browser during an outage. Architect-flagged, fixed in v2.
**How to apply:** any new public marketing page that should work offline must be added to `PUBLIC_PAGES` in sw.js; any new authed page needs nothing (excluded by default).

**Bump `VERSION` in sw.js on every sw.js edit.** Cache names derive from it; activate purges old `arenas-*` caches. Byte-diff triggers the update; skipWaiting+claim make it fast.

**Every new served page needs the PWA head block** (manifest link, theme-color, apple-touch-icon, apple metas, `arenas-pwa.js` deferred) after the viewport meta — pattern-copy from any existing page. Also: install card's logged-in signal is `.bottom-nav` presence in DOM; keys `arenas_install_dismissed` (60-day re-offer) / `arenas_install_done` in localStorage.

**SW treats 502/503/504 as offline-equivalent** (cache → offline-page fallback; 200s/redirects/404/500 always pass through). Deliberate: edge proxies answer 502 when the app is unreachable (mid-deploy), and it is the only network-failure mode testable through the Replit proxy.

## Testing quirks

- Playwright `setOffline` does NOT affect the SW network stack — SW fetches live network and "offline" tests silently pass/fail wrong. Test offline via temp routes that `req.socket.destroy()` — but the proxy converts that to a **502 response**, which is why the gateway-fallback path doubles as the testable offline path.
- Navigation requests have redirect mode `manual`: auth redirects surface in the SW as `opaqueredirect` (status 0) — pass through, never cache (`response.ok && type==='basic'` gate).
- `beforeinstallprompt` can be synthesized in Playwright (`new Event(...)` + stubbed `prompt`/`userChoice`); `window.__arenasPwaState()` debug hook reports eligibility reasons.
- preventDefault on every BIP is safe here: MPA — the event re-fires on each navigation.
